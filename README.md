Kogan.com Engineering & Technology
Welcome to the official GitHub space for the Engineering and Technology team at Kogan.com – Australia's largest pure-play online retailer. This organisation hosts our open source contributions, shared infrastructure components, internal tools, and collaborative projects aimed at empowering innovation across the business.

🚀 Our Mission
We design and deliver scalable, secure, and high-performing technology solutions that power every part of Kogan.com’s ecosystem – from logistics and customer service to data science, payments, and website performance.

🔍 What You'll Find Here

This GitHub enterprise instance serves as the central hub for:
Internal tooling – microservices, automation scripts, CLI utilities, and developer experience enhancements.
Infrastructure as Code (IaC) – Terraform modules, CI/CD pipelines, Kubernetes manifests, etc.
Shared libraries – reusable code for common business logic, integrations, and services.
Open source initiatives – contributions from Kogan.com to the broader tech community.
Proof of Concepts (PoCs) – experimental ideas and technology validations.
Note: Some repositories are private due to commercial sensitivity, regulatory requirements, or internal-only use.

🧩 Repository Structure
Our repositories are prefixed to indicate their purpose:

infra- – Infrastructure as Code
svc- – Microservices
lib- – Shared libraries
tool- – Internal tools and CLIs
poc- – Proof of concepts and experimental work
oss- – Open source projects and community contributions
🔐 Access & Permissions
Access to this organisation is restricted to authorised Kogan.com team members. Please ensure your GitHub account uses 2FA and that you're added via Microsoft Entra as SSO is enforced with additional security monitoring.
To request access, please submit a ticket via our internal ITSM system or contact the Engineering Leadership team.

👩‍💻 Contributing
We follow GitHub Flow for collaboration and encourage all contributors to:

Write clear, testable, and well-documented code
Follow our internal coding guidelines and naming conventions
Use draft pull requests for early feedback
Ensure all commits are signed if working in sensitive or security-related repos
📚 Documentation
Each repository includes its own README.md, usage examples, and architectural overview where applicable. For cross-team standards, please visit the appropriate Wiki.
